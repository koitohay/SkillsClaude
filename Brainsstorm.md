I would like to create a web app for booking different services that satisfies the following requirements:
1. It is a webapp that anybody can start using by providing their email, name and telephone number in Denmark. 
2. Then the person can provide what service he/she is looking for or can search through a list of categories, by default add salon, nails, massage, dentist, kiropractor.
Then they can select the date and time and cities in Denmark.
Then they should be able to review and send the request.
The other side is the service providers, who can see these services coming into them if it is relevant to them. create some dummy services providers. they should create an account select what services do they provide, their name, email , address, telephone number, CVR number. They should be able to offer the service and provide a price for the services.
the client should be able to accept or decline or provide another price as well.
once the service is selected confirmation is sent to both the provider and client by mail.
The project should be created in a proper architecture which supports deployment on Azure. the code should be Infrastructure to code. It should create pieplines to upload the code directly to different environment like dev, test and production.
Everything should be TDD, so there should be unit, integration and UI test created. The UI test should be created in playwright, so all requirement can be tested.
Make sure the the project is able to run locally and can be deployed easily on the cloud.
